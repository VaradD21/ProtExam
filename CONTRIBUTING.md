# Contributing to ProExam

Thanks for your interest in contributing to ProExam! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a feature branch (`git checkout -b feature/amazing-feature`)
4. Make your changes
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Development Setup

```bash
# Install dependencies
npm install

# Create .env file from .env.example
cp .env.example .env

# Start development server
npm run dev
```

## Code Style

- Use meaningful variable and function names
- Add comments for complex logic
- Follow existing code patterns
- Test changes locally before submitting PR

## Reporting Issues

When reporting bugs, please include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser/OS version

## Feature Requests

When requesting features:
- Describe the feature and its use case
- Explain why it would be valuable
- Provide examples if possible

## Testing

Before submitting a PR:
- Test the feature locally
- Check for console errors
- Verify backend responses
- Test on different browsers if UI changes

## Questions?

Feel free to open an issue to ask questions or get clarification.

Thank you for contributing! 🎉
